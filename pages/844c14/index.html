    df指标详解

    使用 df -h 命令可以查看系统中各个文件系统的磁盘使用情况，以下是输出结果中常见指标的解释：

    # 示例输出

    Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   20G   30G   40% /
tmpfs           3.9G  1.2M  3.9G    1% /dev/shm
tmpfs           3.9G  1.6M  3.9G    1% /run
tmpfs           3.9G     0  3.9G    0% /sys/fs/cgroup
/dev/sda2       100G   70G   30G   70% /home
    1
    2
    3
    4
    5
    6

    # 指标解释

    1. Filesystem: 文件系统

      • 表示文件系统的名称或挂载设备的名称。通常是设备文件路径，如 /dev/sda1，或者是逻辑卷名称。

    2. Size: 总大小

      • 表示文件系统的总容量。使用人类可读的格式显示，如 GB（千兆字节）或 TB（太字节）。

    3. Used: 已使用

      • 表示文件系统已使用的空间大小。

    4. Avail: 可用空间

      • 表示文件系统中可供新数据使用的剩余空间大小。

    5. Use%: 使用率

      • 表示文件系统已使用空间占总空间的百分比。

    6. Mounted on: 挂载点

      • 表示文件系统的挂载点，即在目录树中访问该文件系统的位置。

    # 详细解释

    • /dev/sda1: 物理磁盘分区，50G 总容量，其中 20G 已使用，30G 可用，使用率为 40%，挂载在根目录 /。
    • tmpfs: 临时文件系统，通常用于内存中的文件系统。3.9G 总容量，其中 1.2M 已使用，3.9G 可用，使用率为 1%，挂载在 /dev/shm（共享内存目录）。
    • /dev/sda2: 物理磁盘分区，100G 总容量，其中 70G 已使用，30G 可用，使用率为 70%，挂载在 /home。

    # 常见用法

    • 查看所有文件系统的磁盘使用情况:
      df -h
      1
    • 查看特定文件系统的磁盘使用情况:
      df -h /home
      1

    通过 df -h 命令可以方便地查看系统中各个文件系统的磁盘使用情况，有助于磁盘空间管理和监控。
